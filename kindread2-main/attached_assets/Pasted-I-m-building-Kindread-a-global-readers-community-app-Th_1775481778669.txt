FEATURE 1 — FOLLOW SYSTEM

Run this in Supabase SQL editor:

  CREATE TABLE IF NOT EXISTS follows (
    id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    follower_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
    following_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
    created_at  TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(follower_id, following_id)
  );

  CREATE INDEX idx_follows_follower ON follows(follower_id);
  CREATE INDEX idx_follows_following ON follows(following_id);

- isFollowing(followerId, followingId) → boolean

FEATURE 3 — NOTIFICATIONS (the 3 core triggers)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The notifications table already exists. Wire up these 3 Supabase 
database triggers via SQL:

Trigger 1 — Like notification:
  When a row is inserted into bookmark_likes,
  insert into notifications:
    { user_id: bookmark.user_id, actor_id: new.user_id, 
      type: 'like', bookmark_id: new.bookmark_id }
  (skip if actor = bookmark owner — don't notify yourself)

Trigger 2 — Comment notification:
  When a row is inserted into bookmark_comments,
  insert into notifications:
    { user_id: bookmark.user_id, actor_id: new.user_id,
      type: 'comment', bookmark_id: new.bookmark_id }
  (skip if actor = bookmark owner)

Trigger 3 — Session join notification:
  When a row is inserted into club_members,
  insert into notifications:
    { user_id: club.host_id, actor_id: new.user_id,
      type: 'session_join', club_id: new.club_id }
  (skip if the joiner is the host)

In NotificationsScreen, fetch and render these notifications 
in order. Each row shows: avatar, "{name} liked your post / 
commented / joined your session", time ago. 
Tapping a like/comment notification navigates to that bookmark.
Show an unread dot on the bell icon in the header until viewed.
